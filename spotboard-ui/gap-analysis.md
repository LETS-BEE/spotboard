# Gap Analysis: Spotboard Webapp vs Spotboard-UI

이 문서는 기존 `webapp`(Vanilla JS/jQuery)과 새로운 `spotboard-ui`(Vue 3/Nuxt 4) 간의 차이점을 설명하고, 필요한 수정 사항을 요약합니다.

## 1. 풍선 (시각적 피드백)
*   **기존:** 각 팀 행 내의 전용 `.balloons` 컨테이너에 해결된 문제에 대한 풍선 이미지를 쌓아서 표시합니다.
*   **현재 UI:** 구현되지 않음.
*   **조치:**
    *   풍선 에셋을 `public/balloons`로 이동합니다.
    *   `TeamRow.vue`에서 해결된 문제에 대해 풍선을 렌더링하는 로직을 구현합니다.
    *   CSS 스태킹 효과를 복제합니다.

## 2. 최근 이벤트 (타임라인)
*   **기존:** 최근 실행 제출(수락, 대기, 실패)의 타임라인을 보여주는 "대시보드" 사이드바.
*   **현재 UI:** 누락됨.
*   **조치:**
    *   `RecentEvents.vue` 컴포넌트를 생성합니다.
    *   `ContestStore`에서 최근 실행을 추적합니다.
    *   적절한 스타일(색상, 아이콘)로 실행을 표시합니다.

## 3. 컨트롤 및 자동 피드
*   **기존:**
    *   들어오는 실행을 자동으로 처리하는 "자동 피드"(재생/일시정지) 버튼.
    *   해결된 실행에 대한 데스크탑 알림을 위한 "알림" 토글.
    *   하나 피드 / 모두 피드 버튼.
*   **현재 UI:** "하나 피드" 및 "모두 피드" 버튼만 존재함.
*   **조치:**
    *   `ContestStore`에 `isAutoFeeding` 상태 및 타이머 루프를 구현합니다.
    *   `notificationsEnabled` 상태 및 브라우저 알림 API 통합을 구현합니다.
    *   `HeaderControls.vue`를 재생/일시정지 및 알림 토글 버튼으로 업데이트합니다.

## 4. 디자인 및 레이아웃
*   **기존:** 커스텀 CSS(scoreboard.css), 밀집된 정보 레이아웃.
*   **현재 UI:** Material 3 Expressive. 깔끔하지만 정보 밀도가 낮을 수 있음.
*   **조치:** 새로운 컴포넌트(풍선, 최근 이벤트)가 Material Design 시스템에 맞으면서도 기존 스코어보드의 유용성을 유지하도록 합니다.

## 5. 에셋
*   **기존:** `webapp/src/assets/balloons/*.png`
*   **현재 UI:** 이 파일들이 `public/`에 필요함.
*   **조치:** 에셋을 복사합니다.
